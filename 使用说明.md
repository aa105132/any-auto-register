# 逆向分析 Workflow 使用说明

> 目标:用一组逆向工具(抓包 / JS 逆向 / 静态分析 / 动态插桩)配合 AI 客户端做逆向。
> 本说明记录本项目可用的网页逆向工具与用法。

---

## 一、整体架构

```
工具层(已装好)
  ├─ Anything Analyzer 应用   →  MCP 端口 23816,抓包/AI协议逆向(30工具)
  ├─ js-reverse-mcp           →  MCP(stdio),JS 逆向/CDP断点/反检测(22工具)
  ├─ frida CLI                →  动态插桩/Hook(v17.9.4,走 PATH)
  └─ radare2 CLI              →  静态二进制分析(解包/反汇编/补丁,走 PATH)

方法论层(逆向 skill,需要时 LLM 自己读)
  └─ ~/.claude/skills/ 下 5 个:reverse-engineering / radare2 / ida-reverse / apk-reverse / mcp-js-reverse-playbook

AI 客户端层
  ├─ Claude Code   →  原生加载 skills + 项目级 MCP(本目录已配)
  ├─ Codex         →  全局 MCP + 读本目录 AGENTS.md
  └─ ZCode         →  全局 MCP + 读本目录 AGENTS.md
```

## 三、启动 Anything Analyzer 应用(用 anything-analyzer MCP 前必做)

1. 双击 `D:\Anything Analyzer\Anything Analyzer.exe`
2. 设置 → MCP 服务端 → 开启,端口 **23816**(默认,不用改),不设鉴权
3. 设置 → LLM,填 API Key(只用应用内置 `run_analysis` 时才需要)

验证:访问 `http://localhost:23816/mcp` 有响应即正常。

## 五、逆向 workflow 实操

### 场景 A:网站 API / JS 加密逆向

- **抓流量**:anything-analyzer MCP → create_session → start_capture → navigate → 操作 → stop_capture → filter_requests/get_request_detail
- **JS 深挖**:js-reverse MCP → 列脚本/搜代码/设断点/断点求值/导出网络材料/补环境
- **或** 让应用内置 AI 出报告:run_analysis(用第三步配的 API Key)

### 场景 B:App / 终端 / 手机抓包(MITM)

1. 应用 → 设置 → MITM 代理 → 装 CA 证书(首次需管理员)→ 开代理(默认 8888)
2. 配代理:终端 `curl -x http://127.0.0.1:8888 ...`;Python `proxies={...}`;桌面应用一键设系统代理;手机 Wi-Fi 代理 + 装 CA
3. anything-analyzer:create_session(URL 可空)→ start_capture → 外部流量流入 → stop_capture → 分析

### 场景 C:二进制静态分析(PE/ELF/so/dex/wasm)

让 LLM 读 `radare2` skill,用 `r2`/`rabin2` 做侦察/反汇编/看字符串导入/patch。
IDA 工作流则读 `ida-reverse` skill(需 IDA Pro + ida-multi-mcp,已在你全局 MCP 里)。

### 场景 D:动态插桩 / Hook

`frida` 走 PATH。设备端需另跑 `frida-server`。APK 场景的 Frida Hook 流程见 `apk-reverse` skill。

### anything-analyzer 的 30 个工具速查

| 类别 | 工具 |
|------|------|
| 会话 | list_sessions / create_session / start_capture / stop_capture / pause_capture / resume_capture / delete_session |
| 浏览器 | navigate / browser_back / forward / reload / create_tab / close_tab / list_tabs / clear_browser_env / browser_screenshot / cdp_send_command |
| 数据 | get_requests / filter_requests / get_request_detail / get_hooks / get_storage |
| 分析 | run_analysis / get_reports / chat_followup |
| 录制回放 | get_interactions / get_interaction_summary / replay_interactions / execute_browser_action / get_page_elements |

## 七、常见问题

**Q: 客户端报连不上 anything-analyzer MCP?**
A: 应用没开或 MCP 服务端没开。启动应用 → 设置 → MCP 服务端 → 开启(端口 23816)。

**Q: js-reverse MCP 首次很慢?**
A: 首次 `npx js-reverse-mcp` 要从 npm 拉包,等一会;之后就快了。需本机装了 Chrome。

**Q: radare2 `r2 -v` 没反应?**
A: 已装在 `D:\radare2\radare2-6.1.8-w64\bin` 并入用户 PATH。当前终端没生效就**重开终端**刷新 PATH;或临时用全路径 `D:\radare2\radare2-6.1.8-w64\bin\radare2.exe`。

**Q: 换 anything-analyzer 端口?**
A: 应用里改端口后,同步改三处配置的 `23816:`~/.codex/config.toml`、`~/.claude.json`、`~/.zcode/cli/config.json`。

**Q: 卸载逆向环境?**
A: 三处 MCP 配置删 anything-analyzer/js-reverse 节;`~/.claude/skills/` 删 5 个逆向 skill;frida `pip uninstall frida-tools`;radare2 删目录 + 移 PATH;应用从 `D:\Anything Analyzer\Uninstall Anything Analyzer.exe` 卸载。
